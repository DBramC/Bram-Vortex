# Bram Vortex🌀

Bram Vortex is an advanced, AI-powered Cloud Infrastructure Automation Platform, developed as the core deliverable of my **Bachelor's Thesis**.

Designed to bridge the gap between application development and cloud operations, Bram Vortex is a web-based system that intelligently analyzes application repositories to automatically determine, provision, and manage the most suitable cloud infrastructure. By leveraging a modern GitOps philosophy alongside AI agents, the platform enables automated, scalable, and highly secure deployment pipelines.

# Cloud-Native Infrastructure & Kubernetes🐳 

The entire ecosystem is designed to run exclusively on Kubernetes (K8s) ☸️, following fully declarative configuration patterns:

  *🌉 API Gateway / Ingress: External traffic and routing between microservices are handled by the **Kong API Gateway**, which provides centralized control, TLS termination, and service load balancing within the cluster.

  *🗄️ Stateful Data Management: PostgreSQL 🐘 is used for data persistence. To ensure high availability and reliability in K8s, the database is provisioned as a StatefulSet, accompanied by separate Persistent Volume Claims (PVCs) for secure disk storage management.

# Zero-Trust Security & Secrets Management (HashiCorp Vault)🔐

Sensitive data 🛡️ (such as API keys for Google GenAI, DB credentials, and OAuth secrets) are not stored in images or plain text manifests. The system integrates enterprise-grade secrets management and strict access controls:

🏦 It utilizes HashiCorp **Vault** in combination with the Secrets Store CSI Driver of Kubernetes.

🔑 Each microservice has its own defined SecretProviderClass (e.g., auth-svc-secret-provider.yaml, postgres-secret-provider.yaml). This allows K8s to pull encrypted secrets from the Vault and mount them directly to the file system of the pods at runtime.

🎫 Stateless Authentication: System security is reinforced using **JWT (JSON Web Tokens)**. These tokens are exclusively generated by the Authentication Service upon successful user login and are rigorously verified by every other microservice to ensure secure, stateless authorization and inter-service communication.

# CI/CD & GitOps🔄

The deployment pipeline is strictly structured around GitOps principles. 📦

🤖 Automated Continuous Integration (CI) pipelines are orchestrated via GitHub Actions, validating builds and generating new container images.

🐙 Continuous Deployment is managed by Argo CD, which constantly polls the repositories and syncs the K8s state according to the declarative configuration.

# Frontend 💻

The UI is a fully decoupled React/TypeScript Single Page Application (SPA). ⚛️ In the K8s ecosystem, it is deployed as a stateless container and communicates with the backend services exclusively through RESTful API calls routed via the Kong Gateway. 🌐
